# Internationalization (i18n)

## Why i18n?

WoPeD Next supports both English and German. Instead of hardcoding text strings in templates, we use translation keys that map to the correct string for the active language. This means every user-visible label, button, message, and tooltip must go through the i18n system.

## Key Concepts

### 1. Translation Keys

Translations are organized as nested TypeScript objects. Each locale file (`en.ts`, `de.ts`) exports the same structure:

```typescript
// src/i18n/locales/en.ts
export default {
  common: {
    save: 'Save',
    cancel: 'Cancel',
    close: 'Close',
    delete: 'Delete',
  },
  menu: {
    file: 'File',
    new: 'New',
    open: 'Open',
  },
}
```

```typescript
// src/i18n/locales/de.ts
export default {
  common: {
    save: 'Speichern',
    cancel: 'Abbrechen',
    close: 'Schließen',
    delete: 'Löschen',
  },
  menu: {
    file: 'Datei',
    new: 'Neu',
    open: 'Öffnen',
  },
}
```

Keys use **dot notation** to access nested values: `common.save` resolves to `'Save'` in English and `'Speichern'` in German.

### 2. Using Translations in Templates

Use the `$t` function directly in templates:

```vue
<template>
  <button>{{ $t('common.save') }}</button>
  <span>{{ $t('menu.file') }}</span>
</template>
```

`$t` is globally available in all templates — no import needed.

### 3. Using Translations in Script

In `<script setup>`, use the `useI18n` composable to get the `t` function:

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

const saveLabel = computed(() => t('common.save'))

function confirmDelete() {
  if (confirm(t('common.delete') + '?')) {
    // ...
  }
}
</script>
```

### 4. Adding New Translation Keys

When you add a new feature with user-visible text, you must add the key to **both** locale files. If you forget one, the fallback locale (English) will be used.

Steps:

1. Decide on a section and key name (e.g., `editor.zoomIn`)
2. Add the English text in `src/i18n/locales/en.ts`
3. Add the German text in `src/i18n/locales/de.ts`
4. Use `$t('editor.zoomIn')` in your template or `t('editor.zoomIn')` in script

**Important:** Both locale files must have the same key structure. Missing keys will trigger console warnings in development.

### 5. Locale Switching

The project provides a `setLocale` function to change the active language at runtime:

```typescript
import { setLocale } from '@/i18n'

setLocale('de') // switch to German
setLocale('en') // switch to English
```

This updates `i18n.global.locale.value`, and all `$t()` / `t()` calls reactively update throughout the application.

## Project Example

Here's how i18n is set up and used in WoPeD Next:

**Setup (`src/i18n/index.ts`):**

```typescript
import { createI18n } from 'vue-i18n'
import en from './locales/en'
import de from './locales/de'

export const i18n = createI18n({
  legacy: false,
  locale: 'en',
  fallbackLocale: 'en',
  messages: { en, de },
})

export function setLocale(locale: string) {
  i18n.global.locale.value = locale as 'en' | 'de'
}
```

- `legacy: false` enables the Composition API mode (required for `useI18n()`)
- `locale: 'en'` sets English as the default
- `fallbackLocale: 'en'` shows English text when a German translation is missing
- `messages` provides both locale objects

**Usage in a component:**

```vue
<template>
  <div class="toolbar">
    <button @click="handleNew">{{ $t('menu.new') }}</button>
    <button @click="handleOpen">{{ $t('menu.open') }}</button>
    <button @click="handleSave">{{ $t('common.save') }}</button>
  </div>
</template>

<script setup lang="ts">
import { useI18n } from 'vue-i18n'

const { t } = useI18n()

function handleNew() {
  console.log(t('menu.new')) // 'New' or 'Neu'
}
</script>
```

## Exercises

### Starter: Add a Greeting Key

1. Add `common.greeting` with value `'Welcome'` to `src/i18n/locales/en.ts`
2. Add `common.greeting` with value `'Willkommen'` to `src/i18n/locales/de.ts`
3. Create a component that displays `{{ $t('common.greeting') }}`
4. Verify it works by switching locales

### Standard: Translated Component with Language Toggle

Build a component that:

1. Displays 5 translated labels (e.g., a small form with field labels)
2. Includes a button that toggles between English and German
3. Uses `setLocale` for switching
4. All text updates reactively when the language changes

```vue
<template>
  <div>
    <button @click="toggleLocale">
      {{ $t('common.switchLanguage') }}
    </button>
    <!-- your translated labels here -->
  </div>
</template>
```

### Challenge: Editor Tools Translation Section

1. Add a new `editor.tools` section to both locale files with at least 8 tool names (e.g., `select`, `place`, `transition`, `arc`, `token`, `delete`, `undo`, `redo`)
2. Create a toolbar component that renders each tool as a button using `v-for`
3. Each button should display the translated tool name
4. The toolbar should work correctly in both languages
5. Use `useI18n` in the script to access tool names programmatically
